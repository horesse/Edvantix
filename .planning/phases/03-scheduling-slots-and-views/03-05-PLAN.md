**GetScheduleQueryHandler (per D-07, D-08, D-09, D-15):**

Inject: `SchedulingDbContext` (direct context access for complex query), `ITenantContext`, `ClaimsPrincipal` (for profileId), the Organizations gRPC permission client, and `IOrganizationsGroupService`.

The handler implements the following logic:

1. Extract caller's `profileId` from `ClaimsPrincipal`.

2. The endpoint has `.RequireAuthorization(SchedulingPermissions.ViewSchedule)` as baseline gate. Inside the handler, make TWO additional gRPC calls to determine role level:

   **First:** Check if manager (proxy = `scheduling.create-lesson-slot`):
   ```csharp
   var isManager = (await grpcClient.CheckPermissionAsync(...SchedulingPermissions.CreateLessonSlot...)).HasPermission;
   ```

   **Second:** If not manager, check if teacher (via dedicated permission):
   ```csharp
   var isTeacher = !isManager && (await grpcClient.CheckPermissionAsync(...SchedulingPermissions.ViewOwnSchedule...)).HasPermission;
   ```

   Both calls are served from HybridCache so they are cheap.

3. Build the base query — query `LessonSlots`, filter by date range.

4. Apply data filtering based on permission level:
   - **Manager** (`isManager == true`): No additional filter — return ALL tenant's slots. Include `TeacherId`, `TeacherName` (placeholder), `StudentCount` (resolve via Organizations gRPC or set to null in v1).
   - **Teacher** (`isTeacher == true`): Filter to `slot.TeacherId == profileId`. Include `StudentCount`. Set `TeacherId = null`, `TeacherName = null`.
   - **Student** (neither manager nor teacher): Call `IOrganizationsGroupService.GetGroupsForStudentAsync(tenantContext.SchoolId, profileId, ct)` to get the list of GroupIds. Filter to slots where `slot.GroupId IN groupIds`. Include `TeacherId`, `TeacherName`. Set `StudentCount = null`.

5. For each slot, resolve GroupName and GroupColor by calling `IOrganizationsGroupService.GetGroupAsync(slot.GroupId, ct)` (consider caching or batch call to avoid N+1).

6. Return `List<ScheduleSlotDto>`.
