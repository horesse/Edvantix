## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline gaps |
| sm | 8px | Compact element spacing, button icon gap |
| md | 16px | Default element spacing, card inner padding |
| lg | 24px | Section padding, card header padding |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical spacing |

Touch-target exception: Action buttons (Add Credit, Add Adjustment) must meet a minimum interactive element height of 44px per WCAG 2.5.8. This is a minimum height constraint on interactive elements — it is NOT a spacing scale token and must not be used as a `gap` or `padding` value in layouts.

## Typography

Exactly 4 sizes and 2 weights are declared. No other values may be used.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Table cell text, form labels, descriptions |
| Label | 12px | 700 (bold) | 1.4 | Table column headers (uppercase, tracking-wider), metadata, micro-labels |
| Heading | 18px | 700 (bold) | 1.2 | Page heading (matches `text-lg font-bold tracking-tight` from attendance page) |
| Display | 20px | 700 (bold) | 1.1 | KPI stat values (balance remaining, total credits) — uses `text-xl font-bold` |

Declared weights: **400 (regular)** and **700 (bold)** only. The Label role achieves visual distinction from body text through `uppercase` + `tracking-wider` at weight 700 rather than through a third weight value.

Font: `Inter` via `--font-sans`. Numeric values (balance figures, lesson counts) use `tabular-nums` class for alignment in tables and KPI cards.

## Screen Inventory

Three distinct UI surfaces for this phase:

### Surface 1: Student Balance Page (`/school/students/{studentId}/balance`)

Manager-facing page. Shows balance summary KPI cards and the ledger transaction table.

**Primary visual anchor:** The "Остаток" KPI card is the focal point of this surface. It is the only element that changes color based on data state (emerald when positive, rose when zero or negative). All other KPI cards are static in appearance. The executor must render "Остаток" as the third card in the KPI row so it draws the eye before the debt status card.

**Layout:** Vertical stack matching attendance page pattern.

```
Page Header (student name + "Баланс студента" subtitle)
  └── [Add Credit button] [Add Adjustment button]
KPI Row (4 cards, 2×2 on mobile → 4 columns on sm+)
  ├── KpiCard: Всего куплено / {N} уроков
  ├── KpiCard: Использовано / {N} уроков
  ├── KpiCard: Остаток / {N} уроков  ← PRIMARY ANCHOR: emerald when positive, rose when ≤ 0
  └── KpiCard: Статус долга / Есть долг | Нет долга
Ledger Table Card
  └── Table: Date | Type | Amount | Description | Running Balance
```

**Balance remaining KPI:** When `remaining > 0` → emerald icon + value. When `remaining <= 0` → rose icon + value. This is the only place balance sign affects color.

### Surface 2: Add Credit Sheet (Dialog)

Slides in from the right (use `Sheet` from `@workspace/ui/components/sheet` if available, otherwise `Dialog`). Manager adds a positive credit to student's balance.

```
Sheet Title: "Пополнить баланс"
Form:
  └── Input: Количество уроков (number, min 1)
  └── Textarea: Примечание (optional)
Footer: [Отмена] [Пополнить баланс] ← primary CTA
```

### Surface 3: Add Adjustment Sheet (Dialog)

Same sheet pattern. Manager applies a manual credit or debit correction.

```
Sheet Title: "Корректировка баланса"
Form:
  └── Select: Тип — Начислить / Списать
  └── Input: Количество уроков (number, min 1)
  └── Textarea: Примечание (required for Manual Debit)
Footer: [Отмена] [Применить] ← neutral CTA for credit, destructive variant for debit
```

### Surface 4: Payment Status Column (Scheduling Slot Attendance)

Existing `GET /slots/{slotId}/attendance` table gains a `PaymentStatus` column. This is an extension to an existing screen, not a new page.

```
Existing attendance table column addition:
  └── "Оплата" column → Badge per student
      ├── Paid   → Badge variant="default" (emerald bg, green text)
      ├── Unpaid → Badge variant="secondary" (muted)
      └── Debt   → Badge variant="destructive" (rose bg)
```

## Interaction States

### Balance KPI Cards

- Loading: Render 4 `Skeleton` placeholders at same size as KpiCard (`h-16 rounded-xl`)
- Error: Replace cards with a single `alert` (outline variant) spanning full width — error copy above
- Loaded: Static display — no hover state, no click action

### Add Credit / Add Adjustment Buttons

- Default: visible, enabled
- Submitting: Button disabled, `Spinner` replaces leading icon, text remains unchanged
- Success: Sheet closes, toast `"Баланс обновлён"` via `sonner`
- Error: Sheet stays open, toast error below form

### Ledger Table

- Loading: 5 `Skeleton` rows at `h-10` each inside the table body
- Empty: Replace table body with `Empty` component (centered, dashed border, icon + heading + body + CTA button)
- Loaded: Standard table with hover row (`hover:bg-muted/20`)
- Pagination: If > 50 transactions, show `Pagination` component below table

### Payment Status Badge (Scheduling Slot)

- Rendered inline in existing attendance table
- No loading state (value arrives with parent API response)
- No interactive state — read-only display

### Manual Debit Confirmation

- Triggered when user clicks "Списать" in the adjustment sheet
- `AlertDialog` opens (blocks the sheet)
- Confirm button goes to destructive loading state while request in-flight
- On success: AlertDialog closes, Sheet closes, toast `"Списание применено"`
- On error: AlertDialog closes, Sheet stays open, toast error

## Accessibility

- All badge status indicators include `aria-label` matching their text content
- KPI cards use `<dl>` / `<dt>` / `<dd>` semantic structure or `role="region"` with descriptive `aria-label`
- Ledger table uses `<table>` with `<thead>` / `<tbody>` — already enforced by shadcn `Table` component
- Form inputs have associated `<label>` via shadcn `Form` + `Field` components — never placeholder-only
- Destructive `AlertDialog` sets `role="alertdialog"` (built into shadcn component)
- Focus is trapped inside open Sheet and AlertDialog
- Payment status badges in the attendance table do not rely on color alone — text label is always present
