gRPC CheckPermission server endpoint in Organizations with HybridCache (L1 in-memory + L2 Redis), plus a GET /permissions REST endpoint for cache priming.

Purpose: Enable any downstream service to resolve a user's permissions for a school in a single cached gRPC call (ORG-07), with the cache layer preventing redundant DB queries (ORG-08).
Output: Working gRPC server, cached query handler, REST priming endpoint, Redis wired in AppHost.
