   - Return `Unit.Value`

5. `AddAdjustmentCommandHandler.cs`:
   - Same auto-create pattern as AddCredit
   - Map `AdjustmentType.Credit` → `LessonTransactionType.ManualCredit`, `AdjustmentType.Debit` → `LessonTransactionType.ManualDebit`
   - Call `balance.AddTransaction(transactionType, command.Amount, description: command.Description)`

6. `AddAdjustmentCommandValidator.cs`:
   - `RuleFor(x => x.StudentId).NotEmpty()`
   - `RuleFor(x => x.SchoolId).NotEmpty()`
   - `RuleFor(x => x.Amount).GreaterThan(0)`
   - `RuleFor(x => x.Type).IsInEnum()`

   ```csharp
   public sealed record GetBalanceDto(Guid StudentId, int Purchased, int Used, int Remaining);
   ```

9. `GetBalanceQueryHandler.cs`:
   - Inject `IStudentBalanceRepository`
   - Fetch via `GetWithTransactionsAsync(query.StudentId)`
   - If null, return `new GetBalanceDto(query.StudentId, 0, 0, 0)`
   - Compute `purchased` = sum of Amount where Type is `ManualCredit` (PAY-03: "purchased" = manually added lessons)
   - Compute `used` = sum of Amount where Type is `AttendanceDebit`
   - Compute `remaining` = sum of all Credit types (ManualCredit + AttendanceReversal) minus sum of all Debit types (AttendanceDebit + ManualDebit) per D-04
   - Return `new GetBalanceDto(query.StudentId, purchased, used, remaining)`

Create three minimal API endpoints following the Scheduling endpoint pattern (IEndpoint interface with MapEndpoint method).

1. `AddCreditEndpoint.cs`:
   - Route: `POST /v1/students/{studentId:guid}/balance/credit`
   - Authorization: `.RequireAuthorization(PaymentsPermissions.ManageBalance)`
   - Request body record: `AddCreditRequest(int Amount, string? Description)`
   - Handler: read `studentId` from route, `schoolId` from `ITenantContext.SchoolId`, construct `AddCreditCommand(studentId, schoolId, request.Amount, request.Description)`, send via `IMediator`, return `Results.NoContent()`
   - Tags: `"Balance"`

2. `AddAdjustmentEndpoint.cs`:
   - Route: `POST /v1/students/{studentId:guid}/balance/adjustment`
   - Authorization: `.RequireAuthorization(PaymentsPermissions.ManageBalance)`
   - Request body record: `AddAdjustmentRequest(AdjustmentType Type, int Amount, string? Description)`
   - Handler: same pattern as AddCredit but with `AddAdjustmentCommand`
   - Tags: `"Balance"`

3. `GetBalanceEndpoint.cs`:
   - Route: `GET /v1/students/{studentId:guid}/balance`
   - Authorization: `.RequireAuthorization(PaymentsPermissions.ViewBalance)`
   - Handler: read `studentId` from route, send `GetBalanceQuery(studentId)` via `IMediator`, return `Results.Ok(dto)`
   - Tags: `"Balance"`

All endpoints must inject `ITenantContext` for SchoolId (command endpoints) and `IMediator` for dispatch.
